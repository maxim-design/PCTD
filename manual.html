PCTD manual

PCTD -Program Copies for Testing Deployment.

Is a program that was created in order to deploy copies of a selected program in numbered folders for automation testing.

Extract 7zip in desired destination ( highly recommended in the desktop)

Press on PCTD.EXE

First time user

The application opens in the menu tab. The tab will appear empty for a first time user and a configuration needs to be created.

As long as no configuration file exists the program will inform the user appon entry to the “Settings” tab .

Define and create a configuration file

“Folder” button - use it to select the destination directory where all the copies will be created.

“Source File” - This button will require the user to choose where is the program pack.exe or install file is that will be used to create the copies.

“Copies“ - define how many copies of your program you want to create. Recommended number is between 1 - 9.

Side note: the application will allow upto 99 copies, however to many copies is not recommended.

“Name” - define the name of the program you're making copies for testing of.

Press the “Save” button to create the configuration file used by the app or update the existing one.

The application will create a new configuration file

The file will be saved C:\Users\”UserName”\AppData\Local\PCTD\Configuration\PCTDTool.cfg

Updating values:

It is not necessary to reselect all the values to save the updated configuration. You can change any one or all values and the configuration file will be updated upon saving.

You can press “Clear” in order to clear all the fields.

You can press “Load” to load the current configuration settings.

After saving your choices the “Main” tab will be updated and deploy will be possible.

Deploy versions

The deployment is done in the “Menu” tab. After creating the configuration file the application will automatically display them in the current setup section.

.

It is highly recommended to verify the information before pressing on the “Deploy” button. During the deploy process the application will display progress.

*Note

When the application is deploying, it deletes the previous copies, if any exists, and creates New copies based on the settings provided.

It is highly recommended to verify deployment after the process is done.

F.A.Q:

Q: Why is it not recommended to deploy to many copies?

A:it depends on the available memory and resources load of the machine at the moment of deployment. The app loads the installation file in advance into memory to determine its length and that can cause hanging of the GUI app.

It is important to remember that the unpacking is a separate process from the GUI app and will continue regardless in all trials, the Gui unfroze after a few seconds.

Q: Does the app create a log for me to follow?

A: Yes. The logs folder is located at the same location as the configuration file.

Q: Does the app delete the pack / install file after deployment?

A: No. both the source file and its copies in the target directories are saved.
